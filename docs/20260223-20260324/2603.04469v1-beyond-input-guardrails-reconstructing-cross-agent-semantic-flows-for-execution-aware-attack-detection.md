## 摘要
多智能体系统正逐渐成为复杂任务编排的事实标准。然而，其对自主执行和非结构化智能体间通信的依赖引入了严重的风险（如间接提示词注入），这些风险极易绕过传统的输入防护栏。为此，我们提出了 MAScope 框架，将防御范式从静态输入过滤转向执行感知分析。通过提取和重构跨智能体语义流，MAScope 将碎片化的操作原语合成为连续的行为轨迹，从而实现对系统活动的全局视图。我们利用主管大语言模型（Supervisor LLM）来审查这些轨迹，识别数据流违规、控制流偏差和意图不一致等异常。实证评估表明，MAScope 有效地检测了超过十种不同的复合攻击向量，在节点级和路径级端到端攻击检测中分别实现了 85.3% 和 66.7% 的 F1 分数。源代码可在 https://anonymous.4open.science/r/MAScope-71DC 获取。

## Abstract
Multi-Agent System is emerging as the \textit{de facto} standard for complex task orchestration. However, its reliance on autonomous execution and unstructured inter-agent communication introduces severe risks, such as indirect prompt injection, that easily circumvent conventional input guardrails. To address this, we propose \SysName, a framework that shifts the defensive paradigm from static input filtering to execution-aware analysis. By extracting and reconstructing Cross-Agent Semantic Flows, \SysName synthesizes fragmented operational primitives into contiguous behavioral trajectories, enabling a holistic view of system activity. We leverage a Supervisor LLM to scrutinize these trajectories, identifying anomalies across data flow violations, control flow deviations, and intent inconsistencies. Empirical evaluations demonstrate that \SysName effectively detects over ten distinct compound attack vectors, achieving F1-scores of 85.3\% and 66.7\% for node-level and path-level end-to-end attack detection, respectively. The source code is available at https://anonymous.4open.science/r/MAScope-71DC.

#### 1. 核心问题与整体含义（研究动机和背景）
随着大语言模型（LLM）向多智能体系统（MAS）演进，系统能够处理更复杂的长程任务。然而，MAS 的**自主执行**和**非结构化通信**特性带来了全新的安全挑战。传统的“输入防护栏”（Input Guardrails）主要针对单次交互的静态过滤，难以应对**间接提示注入（Indirect Prompt Injection）**、**特权提升**和**跨智能体攻击**。攻击者可以将恶意指令碎片化地隐藏在看似无害的微操作中，这些操作只有在跨智能体、跨时间的完整执行流中才会显现恶意意图。因此，研究重心需要从“静态输入过滤”转向“动态执行感知分析”。

#### 2. 方法论：核心思想与技术细节
论文提出了 **MAScope** 框架，其核心思想是通过重构跨智能体的语义流（Semantic Flows）来识别攻击。主要流程包括：
*   **双层数据采集**：结合应用层（智能体交互、工具调用日志）和内核层（进程、文件、网络事件，如 ETW/eBPF），通过时间戳对齐和实体归一化，构建统一的原始日志。
*   **语义提取与图重构**：
    *   **HSEC 机制**：提出“分层敏感实体约束”（Hierarchical Sensitive Entity Constraint），引导 LLM 将非结构化日志中的敏感信息（如凭据、PII、系统配置）映射到预定义的层级分类中。
    *   **语义图构建**：将交互定义为五元组 $(l, s, r, o, t)$（层、主体、关系、对象、时间戳），构建跨智能体语义图。
    *   **风险评分与路径提取**：根据实体敏感度、操作类型（如 `IP_Send`）和目标可信度计算风险分数，提取高风险的行为轨迹。
*   **轨迹审查（Trajectory Scrutiny）**：利用一个专门的 **Supervisor LLM**，基于三项核心安全策略对重构的路径进行审计：
    1.  **意图一致性**：动作是否偏离用户初始目标。
    2.  **数据流机密性**：敏感实体是否流向未经授权的外部终点。
    3.  **控制流完整性**：是否存在越权操作或异常的特权转移。

#### 6. 主要结论与发现
*   **检测效能显著**：MAScope 在节点级攻击检测中取得了 **85.3%** 的 F1 分数，远高于 VanillaGPT 的 **21.9%**。
*   **语义重构的必要性**：VanillaGPT 往往将孤立的动作视为正常，而 MAScope 通过重构因果链条，能精准识别隐藏在合法操作序列中的恶意意图。
*   **HSEC 的提升作用**：引入分层约束后，敏感信息提取的 F1 分数提升了约 **27%**，有效减少了语义歧义导致的误报。

#### 7. 优点（亮点）
*   **防御范式转移**：从被动的输入过滤转向主动的执行流监控，更符合 MAS 的动态特性。
*   **跨层语义融合**：成功桥接了高层智能体意图与底层系统内核行为之间的“语义鸿沟”。
*   **可解释性强**：Supervisor LLM 不仅给出检测结果，还能引用具体的事件属性作为证据，便于安全审计。
*   **针对性强**：紧扣 OWASP 最新发布的多智能体安全标准，具有很强的实战参考价值。

#### 8. 不足与局限
*   **实时性挑战**：依赖 LLM 对长路径进行审查，可能会带来较高的推理延迟和 API 调用成本，难以满足超大规模实时系统的需求。
*   **策略依赖性**：检测效果高度依赖于 Supervisor LLM 的推理能力和预设策略的完备性，若攻击者能诱导 Supervisor 产生幻觉，防御可能失效。
*   **加密与混淆**：对于高度混淆的代码执行或加密的智能体间通信，提取敏感实体的难度会大幅增加。
*   **环境局限**：实验主要在模拟的 LangGraph 环境中进行，在更复杂的真实生产环境（如涉及大量第三方插件）中的鲁棒性仍需进一步验证。
