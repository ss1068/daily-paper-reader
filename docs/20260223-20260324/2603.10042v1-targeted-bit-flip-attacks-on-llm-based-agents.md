## 摘要
针对性比特翻转攻击 (BFA) 利用硬件故障来操纵模型参数，构成了重大的安全威胁。虽然先前的工作主要针对单步推理模型（如图像分类器），但具有多阶段流水线和外部工具的基于 LLM 的智能体呈现出新的攻击面，且目前尚未得到探索。本研究提出了 Flip-Agent，这是首个针对基于 LLM 智能体的针对性 BFA 框架，能够同时操纵最终输出和工具调用。我们的实验表明，Flip-Agent 在真实世界的智能体任务上显著优于现有的针对性 BFA，揭示了基于 LLM 智能体系统中的一个关键漏洞。

## Abstract
Targeted bit-flip attacks (BFAs) exploit hardware faults to manipulate model parameters, posing a significant security threat. While prior work targets single-step inference models (e.g., image classifiers), LLM-based agents with multi-stage pipelines and external tools present new attack surfaces, which remain unexplored. This work introduces Flip-Agent, the first targeted BFA framework for LLM-based agents, manipulating both final outputs and tool invocations. Our experiments show that Flip-Agent significantly outperforms existing targeted BFAs on real-world agent tasks, revealing a critical vulnerability in LLM-based agent systems.

#### 1. 论文的核心问题与整体含义
*   **研究背景**：基于大语言模型（LLM）的智能体（Agents）正广泛应用于复杂任务。这些系统依赖存储在内存中的模型参数，因此面临硬件故障注入攻击（如 RowHammer）的威胁。
*   **核心问题**：现有的针对性比特翻转攻击（BFA）主要针对图像分类等单步推理模型。而 LLM 智能体具有**多阶段执行流水线**和**外部工具调用**的特性，这使得传统的梯度选择比特方法难以直接应用，同时也产生了此前未被探索的新攻击面。
*   **研究意义**：本文旨在揭示 LLM 智能体在硬件层面的安全脆弱性，并提出首个专门针对智能体架构的 BFA 框架。

#### 2. 论文提出的方法论：Flip-Agent
Flip-Agent 通过操纵模型参数，在受限的比特翻转预算内实现对智能体行为的精准控制。
*   **两大攻击面**：
    1.  **最终输出引导 (Final Output Steering)**：当输入包含特定触发词（Trigger）时，强制智能体生成攻击者预设的最终结果（如特定品牌推荐）。
    2.  **调用操纵 (Invocation Manipulation)**：在不改变最终输出的情况下，强制智能体在中间阶段调用特定的工具或服务（如强制使用某电商平台接口）。
*   **核心技术细节**：
    *   **统一目标函数**：结合了阶段性交叉熵损失（引导生成目标 Token）、清洁数据约束（保持正常输入下的性能）、**注意力增强项**（放大触发词对目标 Token 的影响力）以及**教师强制项**（确保输出格式的一致性）。
    *   **优先级搜索策略 (Prioritized-Search)**：根据梯度幅值将参数分为高影响力和低影响力组。在搜索比特时，优先处理高影响力参数，通过迭代计算比特翻转后的损失降幅，高效识别关键比特位。

#### 6. 论文的主要结论与发现
*   **高攻击效能**：Flip-Agent 在大多数模型上实现了 **90% 以上的 ASR**，同时保持了极高的 CDA（通常在 90% 以上），显著优于传统 BFA 方法。
*   **基准失效原因**：传统方法依赖触发词优化且针对单步模型设计，无法处理智能体长上下文和多阶段逻辑，导致在智能体任务中表现不佳。
*   **防御难度大**：实验表明，即使防御者知道并保护了部分关键比特，攻击者仍能通过翻转其他次优比特实现攻击目标，现有的硬件或软件防御手段难以完全免疫此类攻击。

#### 7. 优点
*   **首创性**：首次系统性地研究了针对 LLM 智能体的比特翻转攻击。

#### 8. 不足与局限
*   **白盒假设**：攻击者需要完全了解模型架构、参数及内存布局，这在某些闭源或高度受限的部署环境下较难实现。
