## 摘要
后缀越狱攻击是大型语言模型（LLMs）红队测试的一种系统性方法，但由于在识别出越狱后缀之前需要评估大量的候选后缀，因此面临极高的计算成本。本文提出了前缀共享 KV 缓存（PSKV），这是一种专为越狱后缀生成量身定制的即插即用推理优化技术。我们的方法源于一个关键观察：在执行后缀越狱时，虽然需要评估大量的候选提示词，但它们都共享相同的目标有害指令作为前缀。因此，PSKV 不再对重复的前缀进行冗余推理，而是为该前缀维护一个单一的 KV 缓存，并将其与每个候选提示词共享，从而以极低的内存开销实现多种后缀的并行推理。这种设计允许采用更激进的批处理策略，而这些策略在其他情况下会受到内存限制。在五个广泛部署的 LLM 上针对六种常用的后缀攻击进行的广泛实验表明，PSKV 在保持原始攻击成功率（ASR）的同时，将推理时间减少了 40%，峰值内存占用减少了 50%。代码已提交并将公开发布。

## Abstract
Suffix jailbreak attacks serve as a systematic method for red-teaming Large Language Models (LLMs) but suffer from prohibitive computational costs, as a large number of candidate suffixes need to be evaluated before identifying a jailbreak suffix. This paper presents Prefix-Shared KV Cache (PSKV), a plug-and-play inference optimization technique tailored for jailbreak suffix generation. Our method is motivated by a key observation that when performing suffix jailbreaking, while a large number of candidate prompts need to be evaluated, they share the same targeted harmful instruction as the prefix. Therefore, instead of performing redundant inference on the duplicated prefix, PSKV maintains a single KV cache for this prefix and shares it with every candidate prompt, enabling the parallel inference of diverse suffixes with minimal memory overhead. This design enables more aggressive batching strategies that would otherwise be limited by memory constraints. Extensive experiments on six widely used suffix attacks across five widely deployed LLMs demonstrate that PSKV reduces inference time by 40\% and peak memory usage by 50\%, while maintaining the original Attack Success Rate (ASR). The code has been submitted and will be released publicly.

#### 1. 核心问题与整体含义
*   **研究背景**：后缀越狱攻击（Suffix Jailbreak Attacks）是评估大语言模型（LLM）安全性的重要手段。这类攻击通过迭代搜索或生成特定的后缀，诱导模型输出有害内容。
*   **核心问题**：现有的攻击方法（如 GCG、BEAST 等）计算成本极高。在每一轮迭代中，需要评估成百上千个候选后缀。虽然这些候选后缀共享相同的有害指令前缀，但标准推理引擎要么重复计算前缀的 KV 缓存（计算冗余），要么在内存中为每个候选后缀复制一份前缀缓存（显存爆炸），这严重限制了批处理规模和搜索效率。
*   **论文意义**：提出了一种名为 **PSKV (Prefix-Shared KV-cache)** 的优化框架，旨在消除冗余计算并降低显存占用，从而显著加速红队测试过程。

#### 2. 方法论
*   **核心思想**：利用“一前缀多后缀”的结构特性，在内存中仅保留一份前缀的 KV 缓存，并通过虚拟广播机制让所有候选后缀共享该缓存。
*   **关键技术细节**：
    *   **层级延迟扩展策略 (Layer-wise Expansion)**：不同于标准实现预先为所有层分配重复内存，PSKV 仅在推理到特定层时，才动态地将单份前缀 KV 向量广播给批次中的所有后缀。计算完成后立即释放临时视图，确保峰值显存仅受单层影响。
    *   **后缀中心对齐 (Suffix-Centric Alignment)**：为了支持不同长度指令的并行化，PSKV 对指令前缀进行左填充（Left-padding），对目标响应进行右填充（Right-padding）。这确保了所有候选后缀在张量中的起始列索引一致，从而实现高效的向量化损失计算和梯度回传。
    *   **梯度兼容性**：与 vLLM 或 SGLang 等高度封装的推理引擎不同，PSKV 保持了对 PyTorch 原生梯度的支持，因此适用于需要反向传播的白盒攻击（如 GCG）。

#### 6. 主要结论与发现
*   **效率提升**：相比于无加速基准，PSKV 平均减少了 **40% 的推理时间**。
*   **显存优化**：相比于标准 KV 缓存复制，PSKV 平均降低了 **50% 的峰值显存占用**。
*   **解决 OOM 问题**：在 BEAST 等高宽度搜索攻击中，标准 KV 缓存常因显存溢出（OOM）失败，而 PSKV 能够平稳运行。
*   **架构敏感性**：PSKV 在采用多头注意力（MHA）的模型（如 Llama-2）上效果比采用分组查询注意力（GQA）的模型（如 Llama-3）更显著，因为 MHA 的 KV 缓存基数更大。

#### 7. 优点
*   **即插即用**：不需要修改模型架构或攻击算法的核心逻辑，易于集成到现有红队工具中。
*   **全能性**：同时支持基于梯度的白盒攻击和基于采样的黑盒攻击。
*   **内存效率极高**：通过层级广播机制，使前缀部分的显存开销与批次大小解耦。

#### 8. 不足与局限
*   **应用场景限制**：该方法专门针对“共享前缀”的后缀攻击。对于不共享前缀的攻击（如针对每个样本生成完全不同提示词的攻击），该方法无效。
*   **边际效应递减**：在词表极大（如 Llama-3, Qwen2.5）或模型采用 GQA 架构时，由于 Logits 张量和激活值占据了显存的大头，KV 缓存优化的相对占比会有所下降。
*   **实验覆盖**：实验主要集中在 7B/8B 规模的模型上，尚未在超大规模模型（如 70B+）上验证其在多卡分布式环境下的表现。
