## 摘要
提示词攻击（包括越狱和提示词注入）对大语言模型（LLM）系统构成了严重的安全性风险。在生产环境中，护栏（guardrails）必须在严格的低延迟约束下缓解这些攻击，这导致了部署上的鸿沟：轻量级分类器和基于规则的系统在分布偏移下难以泛化，而高容量的基于 LLM 的裁判（judges）对于实时执行而言又过于缓慢或昂贵。在本研究中，我们探讨了轻量级通用 LLM 是否能在现实生产约束下可靠地充当安全裁判。通过精心的提示词和输出设计，轻量级 LLM 被引导完成一个结构化的推理过程，包括显式的意图分解、安全信号验证、危害评估和自我反思。我们在一个精选数据集上评估了我们的方法，该数据集结合了来自真实聊天机器人的良性查询以及通过自动化红队测试（ART）生成的对抗性提示词，涵盖了多样化且不断演变的模式。我们的结果表明，通用 LLM（如 gemini-2.0-flash-lite-001）可以作为实时护栏的有效低延迟裁判。该配置目前已作为新加坡公共服务聊天机器人的集中式护栏服务部署在生产环境中。我们还评估了模型混合（MoM）设置，以评估聚合多个 LLM 裁判是否能比单模型裁判提高提示词攻击检测性能，结果仅观察到微小的提升。

## Abstract
Prompt attacks, including jailbreaks and prompt injections, pose a critical security risk to Large Language Model (LLM) systems. In production, guardrails must mitigate these attacks under strict low-latency constraints, resulting in a deployment gap in which lightweight classifiers and rule-based systems struggle to generalize under distribution shift, while high-capacity LLM-based judges remain too slow or costly for live enforcement. In this work, we examine whether lightweight, general-purpose LLMs can reliably serve as security judges under real-world production constraints. Through careful prompt and output design, lightweight LLMs are guided through a structured reasoning process involving explicit intent decomposition, safety-signal verification, harm assessment, and self-reflection. We evaluate our method on a curated dataset combining benign queries from real-world chatbots with adversarial prompts generated via automated red teaming (ART), covering diverse and evolving patterns. Our results show that general-purpose LLMs, such as gemini-2.0-flash-lite-001, can serve as effective low-latency judges for live guardrails. This configuration is currently deployed in production as a centralized guardrail service for public service chatbots in Singapore. We additionally evaluate a Mixture-of-Models (MoM) setting to assess whether aggregating multiple LLM judges improves prompt-attack detection performance relative to single-model judges, with only modest gains observed.

### 论文总结：基于 LLM-as-a-Judge 与模型混合（Mixture-of-Models）的提示词攻击检测

#### 1. 核心问题与整体含义（研究动机和背景）
随着大语言模型（LLM）向具备规划和工具调用能力的“智能体（Agent）”演进，提示词攻击（如越狱和间接提示词注入）的风险显著增加。现有的防御手段存在**部署鸿沟**：
*   **轻量级编码器分类器**（如基于 DeBERTa 的模型）：速度快但推理能力弱，难以识别复杂的“灰色地带”攻击，且容易产生误报。
*   **高容量 LLM 裁判**：识别精度高但延迟大、成本高，难以满足生产环境的实时性要求。
本研究旨在探索如何利用**轻量级通用 LLM**，通过结构化提示词设计，在满足低延迟约束的同时实现高可靠的安全判定。

#### 2. 方法论：核心思想与关键技术
论文提出了 **LLM-as-a-Judge** 框架，将安全检测视为一个结构化的推理任务，而非简单的二分类：
*   **框架剥离（Framing Stripping）**：引导模型忽略学术、虚构或代码化的包装，直接提取核心意图。
*   **分类引导推理（Taxonomy-Guided Reasoning）**：将输入划分为工作流指令、事实概述、安全知识、详细伤害方法和安全绕过尝试等类别。
*   **结构化思维链（Structured CoT）与自我反思**：
    1.  **初步分类**：给出初步类别和伤害等级。
    2.  **强制自我反思**：模型必须生成一段文字来质疑自己的初步判断（例如：这是否可能只是一个带有技术词汇的合法请求？）。
    3.  **最终裁决与置信度**：基于反思给出最终布尔值和置信度评分。
*   **模型混合（MoM）**：采用并行架构，通过加权线性组合多个 LLM 的评分，并利用网格搜索（Grid Search）优化权重和决策阈值。

    *   **对抗样本（159个）**：通过自动化红队工具（如 PAIR）生成，涵盖社会工程、代码混淆、角色扮演等最新攻击模式。

#### 6. 主要结论与发现
*   **轻量级 LLM 的可行性**：经过精心设计的轻量级模型（如 Gemini-2.0-flash-lite）在 F1 分数上超过了专用的安全模型和商业护栏方案，且延迟保持在 1.5 秒左右，适合实时部署。
*   **结构化推理胜过直接判定**：强制模型进行中间步骤推理和自我反思，显著提升了检测的稳定性，优于 OpenAI Guardrails 的直接判定模式。
*   **“思考模式”并非万能**：在提示词已经足够明确的情况下，开启模型的内部推理（Thinking Mode）反而可能导致性能下降。
*   **MoM 收益递减**：模型混合并不总能提升性能，且随着模型数量增加，冲突信号可能增多。提升主要集中在少数互补的模型组合中（如 GPT-5.1 + GPT-5-mini）。

#### 7. 优点与亮点
*   **实战导向**：方法已在新加坡政府服务中实际部署，具有极强的工程实践意义。
*   **推理结构设计**：通过“框架剥离”和“自我反思”有效解决了 LLM 容易被对抗性包装欺骗的问题。
*   **低成本高效率**：证明了无需昂贵的高容量模型或复杂的微调，仅靠 Prompt 优化即可实现高性能护栏。

#### 8. 不足与局限
*   **数据集规模有限**：929 个样本虽然质量高，但难以覆盖所有潜在的攻击变体。
*   **随机性风险**：尽管设置温度为 0，LLM 裁判仍存在一定的随机性，可能导致判定结果在极端情况下不一致。
*   **MoM 复杂性**：模型混合带来的微小提升（约 1-2%）可能无法抵消其带来的额外计算成本和系统复杂性。
