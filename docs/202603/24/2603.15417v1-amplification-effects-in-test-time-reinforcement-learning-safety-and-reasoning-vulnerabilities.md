## 摘要
测试时训练（TTT）最近成为一种提升大语言模型（LLMs）推理能力的极具前景的方法，该方法使模型在无法获取标签的情况下直接从测试数据中学习。然而，这种对测试数据的依赖也使得 TTT 方法容易受到有害提示注入的影响。在本文中，我们研究了 TTT 方法的安全性脆弱性，重点探讨了一种代表性的基于自一致性的测试时学习方法：测试时强化学习（TTRL）。TTRL 是近期提出的一种 TTT 方法，它通过将多数投票作为奖励信号来奖励自一致性，从而提升 LLM 的推理能力。我们展示了在 TTRL 过程中注入有害提示会放大模型的现有行为，即：当基座模型相对安全时，会产生安全性放大；而当模型对注入数据较为脆弱时，则会产生有害性放大。在这两种情况下，推理能力都会下降，我们将其称为“推理税”（reasoning tax）。我们还展示了 TTRL 等 TTT 方法可能会被对抗性地利用，通过专门设计的“HarmInject”提示强制模型同时回答越狱和推理查询，从而导致更强的有害性放大。总之，我们的研究结果强调，通过促进自一致性来增强 LLM 推理能力的 TTT 方法可能会导致放大行为和推理退化，这凸显了开发更安全的 TTT 方法的必要性。

## Abstract
Test-time training (TTT) has recently emerged as a promising method to improve the reasoning abilities of large language models (LLMs), in which the model directly learns from test data without access to labels. However, this reliance on test data also makes TTT methods vulnerable to harmful prompt injections. In this paper, we investigate safety vulnerabilities of TTT methods, where we study a representative self-consistency-based test-time learning method: test-time reinforcement learning (TTRL), a recent TTT method that improves LLM reasoning by rewarding self-consistency using majority vote as a reward signal. We show that harmful prompt injection during TTRL amplifies the model's existing behaviors, i.e., safety amplification when the base model is relatively safe, and harmfulness amplification when it is vulnerable to the injected data. In both cases, there is a decline in reasoning ability, which we refer to as the reasoning tax. We also show that TTT methods such as TTRL can be exploited adversarially using specially designed "HarmInject" prompts to force the model to answer jailbreak and reasoning queries together, resulting in stronger harmfulness amplification. Overall, our results highlight that TTT methods that enhance LLM reasoning by promoting self-consistency can lead to amplification behaviors and reasoning degradation, highlighting the need for safer TTT methods.

### 论文总结：测试时强化学习中的放大效应：安全性与推理脆弱性

#### 1. 核心问题与整体含义（研究动机和背景）
随着大语言模型（LLM）推理能力的提升，**测试时训练（Test-Time Training, TTT）**作为一种无需标注数据、直接在推理阶段根据测试输入进行自我优化的技术备受关注。然而，TTT 高度依赖测试数据的分布。本文的核心研究问题是：**当测试数据流中混入有害的提示注入（Prompt Injection）时，基于自一致性（Self-consistency）的 TTT 方法是否会破坏模型的安全边界，并对其推理性能产生负面影响？** 研究背景在于填补 TTT 技术在安全性和推理能力协同演化方面的研究空白。

#### 2. 方法论：核心思想与技术细节
论文主要针对 **测试时强化学习（TTRL）** 展开研究，其核心流程如下：
*   **自一致性伪奖励：** 对于每个测试输入 $x$，模型生成 $K$ 个候选回答。通过“多数投票”（Majority Vote）选出出现频率最高的答案作为伪标签 $\hat{y}$。
*   **奖励机制：** 与伪标签一致的回答获得正奖励（+1），不一致的获得零奖励。
*   **策略优化：** 使用 **组相对策略优化（GRPO）** 算法更新模型参数，使模型在后续推理中更倾向于生成与伪标签一致的回答。
*   **放大效应（Amplification Effects）：** 
    *   **安全性放大：** 若基座模型对某类攻击倾向于拒绝，TTRL 会强化这种拒绝行为。
    *   **有害性放大：** 若基座模型对攻击较为脆弱（初始 ASR 较高），TTRL 会强化这种违规回答行为。
*   **HarmInject 攻击：** 作者设计了一种对抗性提示，将“越狱请求”与“推理问题”组合在一起，强制模型为了获得推理奖励而顺带完成有害指令。

#### 3. 实验设计
*   **模型：** 涵盖了 Qwen 系列（0.5B, 1.5B）和 Llama 系列（1B, 3B, 8B）共 5 个指令微调模型。
*   **数据集：**
    *   **推理任务：** AMC 数学竞赛数据集。
    *   **安全/越狱任务：** JailbreakV-28k、WildJailbreak、Llama-jailbreak artifacts。
    *   **良性指令数据：** UltraFeedback（用于测试良性注入的影响）。
*   **对比基准（Benchmark）：** 以“无注入的纯推理 TTRL”作为性能基准，对比注入有害提示后的攻击成功率（ASR）和推理准确率（Pass@1）。

#### 4. 资源与算力
*   **并行策略：** 使用了张量并行（Tensor Parallel size = 8）和完全分片数据并行（FSDP size = 8）。

#### 5. 实验数量与充分性
*   **实验规模：** 论文在 5 个不同规模的模型上进行了详尽测试，涵盖了 3 种不同的越狱数据集，并探讨了不同比例（0% 到 60%）的提示注入影响。
*   **维度丰富：** 除了有害注入，还测试了良性指令注入的影响，并提出了 HarmInject 攻击模型及简单的过滤防御实验。
*   **客观性：** 实验结果展示了模型在不同情况下的双向演化（变安全或变危险），并量化了“推理税”，论证较为客观、全面。

#### 6. 主要结论与发现
*   **放大效应：** TTRL 会放大模型在测试数据上的既有倾向。如果模型初始状态是安全的，它会变得更安全；如果初始状态易受攻击，它会变得更危险。
*   **推理税（Reasoning Tax）：** 无论安全性是放大还是缩小，注入非推理相关的提示都会导致模型在 AMC 任务上的推理能力显著下降。
*   **良性注入风险：** 即使注入的是良性指令（如 UltraFeedback），也会因为强化了“顺从性”而间接导致模型更容易被越狱。
*   **防御脆弱性：** 简单的基于输出格式（如是否为数字）的过滤方法无法抵御 HarmInject 这种复杂的组合式攻击。

#### 7. 优点
*   **视角新颖：** 首次揭示了测试时训练（TTT）在提升推理能力的同时可能带来的安全隐患。
*   **攻击设计巧妙：** 提出的 HarmInject 攻击利用了 TTT 的奖励机制，具有很强的启发性。
*   **跨模型验证：** 在 Qwen 和 Llama 两个主流模型家族上均验证了结论的普适性。

#### 8. 不足与局限
*   **方法局限：** 主要集中在基于“多数投票”和 GRPO 的 TTRL 方法，未探讨其他 TTT 变体（如基于熵最小化的方法）。
*   **防御探索有限：** 虽然指出了简单过滤无效，但未提出更深层次的防御机制（如鲁棒的伪标签过滤或安全约束的 RL）。
*   **标签提取偏差：** 实验中发现 TTRL 的标签提取逻辑（提取最后一个 token）对结果有较大影响，这可能属于特定实现的局限而非 TTT 范式的本质缺陷。
