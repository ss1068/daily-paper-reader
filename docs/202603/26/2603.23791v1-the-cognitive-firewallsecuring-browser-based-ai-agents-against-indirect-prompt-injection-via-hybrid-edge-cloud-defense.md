## 摘要
将大语言模型（LLMs）部署为自主浏览器智能体，会以间接提示注入（IPI）的形式暴露显著的攻击面。基于云的防御可以提供强大的语义分析，但会引入延迟并引发隐私担忧。我们提出了“认知防火墙”（Cognitive Firewall），这是一种三阶段拆分计算架构，将安全检查分布在客户端和云端。该系统由本地视觉哨兵（Sentinel）、基于云的深度规划器（Deep Planner）以及在执行时强制执行策略的确定性守卫（Guard）组成。在 1,000 个对抗性样本中，仅靠边缘侧的防御无法检测到 86.9% 的语义攻击。相比之下，完整的混合架构将总攻击成功率（ASR）降低到 1% 以下（静态评估下为 0.88%，自适应评估下为 0.67%），同时保持对产生副作用的操作的确定性约束。通过在本地过滤表示层攻击，该系统避免了不必要的云端推理，并比纯云端基准实现了约 17,000 倍的延迟优势。这些结果表明，在执行边界进行的确定性强制执行可以补充概率性语言模型，且拆分计算为保护交互式 LLM 智能体提供了实用的基础。

## Abstract
Deploying large language models (LLMs) as autonomous browser agents exposes a significant attack surface in the form of Indirect Prompt Injection (IPI). Cloud-based defenses can provide strong semantic analysis, but they introduce latency and raise privacy concerns. We present the Cognitive Firewall, a three-stage split-compute architecture that distributes security checks across the client and the cloud. The system consists of a local visual Sentinel, a cloud-based Deep Planner, and a deterministic Guard that enforces execution-time policies. Across 1,000 adversarial samples, edge-only defenses fail to detect 86.9% of semantic attacks. In contrast, the full hybrid architecture reduces the overall attack success rate (ASR) to below 1% (0.88% under static evaluation and 0.67% under adaptive evaluation), while maintaining deterministic constraints on side-effecting actions. By filtering presentation-layer attacks locally, the system avoids unnecessary cloud inference and achieves an approximately 17,000x latency advantage over cloud-only baselines. These results indicate that deterministic enforcement at the execution boundary can complement probabilistic language models, and that split-compute provides a practical foundation for securing interactive LLM agents.

随着基于大语言模型（LLM）的自主浏览器智能体（Agent）兴起，它们在处理网页内容时面临严重的**间接提示注入（Indirect Prompt Injection, IPI）**威胁。由于智能体将用户指令（控制平面）与外部网页内容（数据平面）放在同一个上下文窗口中处理，攻击者可以通过在网页中嵌入不可见的恶意指令来劫持智能体。
*   **研究动机**：现有的防御手段面临权衡——纯云端防御语义分析强但延迟高且有隐私风险；纯边缘防御速度快但难以识别复杂的语义攻击。
*   **核心目标**：构建一种既能保持低延迟响应，又能提供深度安全保障的混合防御架构。

#### 2. 论文提出的方法论
论文提出了**认知防火墙（Cognitive Firewall）**，这是一种基于**拆分计算（Split-compute）**的三阶段“防御漏斗”架构：
*   **第一层：边缘哨兵（Edge Sentinel）**：运行在浏览器本地（利用 Chrome 内置的 Gemini Nano 模型）。它通过检查 DOM 和 CSSOM 属性（如透明度、字体大小、位置等）来识别视觉欺骗攻击（如零像素文本），在毫秒级内拦截表现层攻击。
*   **第二层：云端深度规划器（Deep Planner）**：对于通过第一层检查的内容，发送至云端（如 Llama 3 8B）。它扮演“安全分析师”角色，识别复杂的语义陷阱、角色扮演攻击或指令覆盖尝试。
*   **第三层：执行守卫（Origin Guard）**：部署在客户端的确定性拦截器。它通过同步 JavaScript 拦截 API 调用（如 `fetch`），根据预设的白名单和意图策略（如只读任务禁止 POST 请求）强制执行安全约束。

#### 6. 论文的主要结论与发现

#### 7. 优点
*   **分层防御思想**：将低成本、确定性的检查放在边缘，高成本、概率性的语义分析放在云端，平衡了性能与安全。

*   **隐私保护**：通过本地预过滤，部分恶意或敏感内容无需上传云端即可被拦截。

#### 8. 不足与局限
*   **视觉分析局限**：目前的哨兵主要基于 DOM/CSS 启发式规则，无法识别图像或二维码中的视觉提示注入（需引入多模态模型）。
