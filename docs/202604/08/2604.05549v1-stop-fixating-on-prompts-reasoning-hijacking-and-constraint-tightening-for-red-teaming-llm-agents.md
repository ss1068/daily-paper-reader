## 摘要
随着基于大语言模型（LLM）的智能体在各个领域的广泛应用，其复杂性带来了新的安全威胁。现有的红队测试方法大多依赖于修改用户提示词，这不仅缺乏对新数据的适应性，还可能影响智能体的性能。为了应对这一挑战，本文提出了 JailAgent 框架，该框架完全避免了对用户提示词的修改。具体而言，它通过三个关键阶段隐式地操纵智能体的推理轨迹和记忆检索：触发器提取、推理劫持和约束收紧。通过精确的触发器识别、实时自适应机制和优化的目标函数，JailAgent 在跨模型和跨场景环境中表现出了卓越的性能。

## Abstract
With the widespread application of LLM-based agents across various domains, their complexity has introduced new security threats. Existing red-team methods mostly rely on modifying user prompts, which lack adaptability to new data and may impact the agent's performance. To address the challenge, this paper proposes the JailAgent framework, which completely avoids modifying the user prompt. Specifically, it implicitly manipulates the agent's reasoning trajectory and memory retrieval with three key stages: Trigger Extraction, Reasoning Hijacking, and Constraint Tightening. Through precise trigger identification, real-time adaptive mechanisms, and an optimized objective function, JailAgent demonstrates outstanding performance in cross-model and cross-scenario environments.

### 2. 方法论：JailAgent 框架
JailAgent 通过三个核心阶段实现隐式操纵：

1.  **触发器提取 (Trigger Extraction)**：
    *   **语法对齐**：利用 BERT 的 WordPiece 分词和 spaCy 的依存句法分析，将输入文本划分为语义单元（短语级）。

2.  **推理劫持 (Reasoning Hijacking)**：
    *   **数据工厂**：自动合成包含（上下文、毒药响应、良性响应）的三元组训练数据。
    *   **重排序器 (Reranker)**：构建一个轻量级的评分模型（基于 SentenceTransformer 编码器和 MLP 评分头）。通过成对排序损失（Pairwise Ranking Loss）进行微调，使模型在推理时倾向于从多个候选输出中选择带有偏见的、被劫持的推理路径。

3.  **约束收紧 (Constraint Tightening)**：
    *   通过联合优化四个损失函数来精炼触发器嵌入（Embedding）：
        *   **特殊性损失 (Particularity Loss)**：使触发器远离良性数据簇。
        *   **聚类损失 (Clustering Loss)**：确保触发器内部令牌的语义紧凑性。
        *   **可分离性损失 (Separability Loss)**：提高在向量检索任务中的命中率。
        *   **边际损失 (Margin Loss)**：扩大毒药条目与良性条目之间的决策边界。
