## 摘要
调用工具的 LLM 智能体可以读取私有数据、调用外部服务并触发现实世界的动作，从而在工具执行点产生安全问题。我们识别出一种拒绝反馈泄露模式，并将其称为“因果洗白”（causality laundering）。在这种模式中，攻击者探测受保护的操作，从拒绝结果中学习，并通过随后看似无害的工具调用将推断出的信息外泄。这种攻击无法仅通过扁平的溯源追踪（flat provenance tracking）来捕获，因为泄露的信息源于被拒绝操作的因果影响，而非直接的数据流。我们提出了智能体引用监视器（Agentic Reference Monitor, ARM），这是一个运行时强制执行层，通过咨询涵盖工具调用、返回数据、字段级溯源和被拒绝操作的溯源图来调解每一次工具调用。ARM 通过完整性格（integrity lattice）传播信任，并使用来自被拒绝操作节点的反事实边（counterfactual edges）来增强图，从而实现对传递性数据依赖和拒绝诱导的因果影响的强制执行。在针对三个代表性攻击场景的受控评估中，ARM 拦截了扁平溯源基准线所遗漏的因果洗白、传递性污点传播以及混合溯源字段误用，同时仅增加了亚毫秒级的策略评估开销。这些结果表明，拒绝感知的因果溯源是保障调用工具的智能体系统安全的一种有用抽象。

## Abstract
Tool-calling LLM agents can read private data, invoke external services, and trigger real-world actions, creating a security problem at the point of tool execution. We identify a denial-feedback leakage pattern, which we term causality laundering, in which an adversary probes a protected action, learns from the denial outcome, and exfiltrates the inferred information through a later seemingly benign tool call. This attack is not captured by flat provenance tracking alone because the leaked information arises from causal influence of the denied action, not direct data flow. We present the Agentic Reference Monitor (ARM), a runtime enforcement layer that mediates every tool invocation by consulting a provenance graph over tool calls, returned data, field-level provenance, and denied actions. ARM propagates trust through an integrity lattice and augments the graph with counterfactual edges from denied-action nodes, enabling enforcement over both transitive data dependencies and denial-induced causal influence. In a controlled evaluation on three representative attack scenarios, ARM blocks causality laundering, transitive taint propagation, and mixed-provenance field misuse that a flat provenance baseline misses, while adding sub-millisecond policy evaluation overhead. These results suggest that denial-aware causal provenance is a useful abstraction for securing tool-calling agent systems.

### 1. 核心问题与整体含义（研究动机和背景）
*   **核心问题**：论文识别并命名了一种名为**“因果洗白”（Causality Laundering）**的攻击模式。
*   **研究动机**：现有的智能体安全防御（如污点追踪、数据流分析）主要关注“成功执行”的工具调用。然而，当一个敏感操作被安全策略**拒绝（Deny）**时，智能体仍能通过“拒绝反馈”推断出敏感信息（例如：文件存在但无权访问）。攻击者可以利用这种推断出的知识，通过后续看似合法的工具调用将信息外泄。
*   **整体含义**：传统的扁平溯源追踪无法捕获这种泄露，因为信息是通过“因果影响”而非“直接数据流”传播的。论文旨在填补这一防御空白。

### 2. 方法论：核心思想与关键技术
论文提出了 **ARM（Agentic Reference Monitor，智能体引用监视器）**，其核心思想是将“被拒绝的操作”视为溯源图中的一等公民。

*   **核心思想**：通过构建一个包含“反事实边”（Counterfactual Edges）的溯源图，追踪拒绝事件对后续操作的潜在因果影响。
*   **关键技术细节**：
    *   **完整性格（Integrity Lattice）**：定义了五个信任等级（系统指令 > 用户输入 > 信任工具 > 不信任工具 > 工具描述），信任度在图中按最小值原则传播。
    *   **溯源图（Provenance Graph）**：包含四类节点（调用、数据、字段、拒绝动作）和四类边（直接输出、输入、反事实、字段所属）。
    *   **反事实边**：当一个调用被拒绝后，ARM 会自动在拒绝节点与紧随其后的下一个工具调用之间建立一条“反事实边”，标记潜在的因果关联。
    *   **四层策略管道**：
        1.  **L1 硬边界**：检查负载大小、敏感路径、凭据模式等。
        2.  **L2 图感知溯源**：核心层，通过图遍历计算最小信任度并检测反事实链。
        3.  **L3 模式衍生约束**：根据工具的 JSON Schema 自动生成防御规则。
        4.  **L4 手动策略**：执行操作员定义的权限令牌（Capability Tokens）。

### 3. 实验设计
*   **实验场景**：设计了三个具有代表性的攻击场景：
    1.  **因果洗白**：探测敏感文件被拒后，通过邮件发送“文件存在”的推断。
    2.  **传递性污点链**：通过中间工具（如格式化工具）多次转换数据，试图洗掉原始的不信任来源。
    3.  **混合溯源字段误用**：利用结构化输出中受信任字段和不受信任字段的混合（如受信任的姓名+不受信任的邮箱）。
*   **对比方法（Benchmark）**：**扁平溯源基准线（Flat Provenance Baseline）**，即仅检查参数值是否出现在历史记录中，而不追踪复杂的因果或字段级关系。

### 4. 资源与算力
*   **硬件环境**：实验在配备 **Apple M4 Pro 芯片、48 GB RAM** 的设备上运行。
*   **算力消耗**：论文未提及 GPU 训练时长，因为 ARM 是一个**运行时（Runtime）监控层**，不涉及模型训练。其策略评估开销极低，中位延迟小于 **1 毫秒**，相对于 LLM 推理时间（100ms-10s）几乎可以忽略不计。

*   **实验规模**：进行了 3 组针对性攻击场景的受控差异化实验，以及 45 组单元测试（涵盖溯源图构建、信任传播、端到端场景等）。

    *   **客观性**：实验采用了“受控差异化”方法，即除了溯源层外，其他策略层完全一致，确保了结果的对比公平性。但实验目前局限于原型验证，缺乏大规模自动化基准测试（如 AgentDojo）的覆盖。

### 6. 主要结论与发现
*   **拒绝感知是必要的**：仅仅阻止敏感操作是不够的，必须追踪“拒绝”这一事实对智能体后续行为的影响。
*   **ARM 的有效性**：ARM 成功拦截了所有三个场景中的攻击，而基准线方法全部失败。
*   **性能可行性**：基于 Rust 后端（rustworkx）的图运算非常高效，能够支持实时在线监控。
*   **确定性防御**：ARM 的决策是确定性的，不依赖于可能被攻击的 LLM 自身进行安全判断。

### 7. 优点
*   **视角独特**：首次形式化了“因果洗白”攻击，指出了现有防御在处理“拒绝反馈”时的逻辑漏洞。
*   **细粒度追踪**：支持字段级（Field-level）溯源，能够处理复杂的 JSON 结构化数据。
*   **低开销**：亚毫秒级的延迟使其非常适合部署在生产环境的工具调用边界。
*   **防御深度**：四层管道设计提供了从简单模式匹配到复杂因果推理的多重保障。

### 8. 不足与局限
*   **启发式假设**：反事实边目前基于“时间邻近性”建立（即拒绝后的下一个调用），这可能导致**误报**（如果后续调用是无关的）或**漏报**（如果攻击者故意延迟外泄步骤）。
*   **依赖人工标注**：字段级的信任覆盖（Field-level overrides）需要集成者手动指定，否则会退化为对象级追踪。
*   **评估范围有限**：尚未在多智能体协作场景或大规模真实攻击数据集上进行广泛测试。
*   **非完全不干扰**：论文承认 ARM 无法完全消除所有隐蔽信道（如时间信道），只能缓解最显著的因果推理泄露。
