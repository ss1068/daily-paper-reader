## 摘要
大语言模型（LLM）及其应用（如智能体）极易受到提示词注入攻击。最先进的提示词注入检测方法存在以下局限性：（1）其有效性随上下文长度的增加而显著下降；（2）缺乏定义何为提示词注入的明确规则，导致检测决策具有隐蔽性、不透明且难以推理。在本研究中，我们提出了 AgentWatcher 来解决上述两个局限性。为了解决第一个局限性，AgentWatcher 将 LLM 的输出（例如智能体的动作）归因于一小组具有因果影响力的上下文片段。通过将检测重点放在相对较短的文本上，AgentWatcher 可以扩展到长上下文。为了解决第二个局限性，我们定义了一组规则，规定了哪些内容构成或不构成提示词注入，并使用一个监控 LLM 根据归因文本对这些规则进行推理，从而使检测决策更具可解释性。我们在工具使用智能体基准测试和长上下文理解数据集上进行了全面评估。实验结果表明，AgentWatcher 能够有效检测提示词注入，并在无攻击情况下保持效用。代码可在 https://github.com/wang-yanting/AgentWatcher 获取。

## Abstract
Large language models (LLMs) and their applications, such as agents, are highly vulnerable to prompt injection attacks. State-of-the-art prompt injection detection methods have the following limitations: (1) their effectiveness degrades significantly as context length increases, and (2) they lack explicit rules that define what constitutes prompt injection, causing detection decisions to be implicit, opaque, and difficult to reason about. In this work, we propose AgentWatcher to address the above two limitations. To address the first limitation, AgentWatcher attributes the LLM's output (e.g., the action of an agent) to a small set of causally influential context segments. By focusing detection on a relatively short text, AgentWatcher can be scalable to long contexts. To address the second limitation, we define a set of rules specifying what does and does not constitute a prompt injection, and use a monitor LLM to reason over these rules based on the attributed text, making the detection decisions more explainable. We conduct a comprehensive evaluation on tool-use agent benchmarks and long-context understanding datasets. The experimental results demonstrate that AgentWatcher can effectively detect prompt injection and maintain utility without attacks. The code is available at https://github.com/wang-yanting/AgentWatcher.

#### 1. 核心问题与整体含义（研究动机和背景）
随着大语言模型（LLM）智能体（Agents）在现实场景中的广泛应用，**提示词注入（Prompt Injection）**攻击成为了严重的安全威胁。攻击者通过在外部上下文（如邮件、网页）中嵌入恶意指令，诱导智能体执行未经授权的操作（如转账、泄露隐私）。
现有的检测防御方法存在两大核心痛点：
*   **长上下文失效**：随着输入文本变长，检测器的准确率和效率显著下降。
*   **决策黑盒化**：现有方法多依赖隐式表示或简单假设，缺乏明确的逻辑规则，导致安全决策不可解释、难以调试。

#### 2. 方法论：核心思想与技术细节
AgentWatcher 提出了一种两阶段的监控框架，旨在实现高效且可解释的防御：

*   **第一阶段：因果上下文归因（Context Attribution）**
    *   **核心思想**：不检测全部长文本，而是只检测对智能体当前动作（Action）产生因果影响的关键片段。
    *   **技术细节**：利用归因 LLM 的注意力权重，计算上下文 Token 对输出动作的平均贡献。
    *   **滑动窗口机制**：识别“汇聚标记”（Sink Tokens，即接收极高注意力的标记），通过滑动窗口评分函数 $S(i)$ 定位影响力最大的区域，并向左右扩展（$w_l, w_r$）以保持指令完整性。最终提取 $K$ 个非重叠的高分窗口作为待检文本。

*   **第二阶段：基于规则的推理检测（Rule-based Detection）**
    *   **核心思想**：引入一个专门的“监控 LLM”（Monitor LLM），根据一组显式的安全规则对归因文本进行逻辑推理。
    *   **规则设计**：定义了 10+ 条规则（如“是否试图绕过原始任务”、“是否恶意控制资金”等），支持手动定制或通过 LLM 自动生成。
    *   **微调优化**：采用 **GRPO（组相对策略优化）** 算法对监控模型进行强化学习微调，使用基于 BLEU 分数的奖励函数，鼓励模型不仅给出“是/否”的判断，还能准确提取出恶意指令并引用相关规则。

#### 6. 主要结论与发现
*   **卓越的防御性能**：在所有基准测试中，AgentWatcher 始终能将 ASR 降低至 10% 以下（多数情况下接近 0%），同时 Utility 损失极小（通常 < 4%）。
*   **长文本扩展性**：通过归因机制，AgentWatcher 在处理长达数万 Token 的上下文时，性能不会像基线方法那样大幅崩溃。
*   **可解释性价值**：监控模型生成的推理过程能清晰指出违反了哪条规则，有助于安全审计。
*   **规则自动化的可行性**：实验证明，由 LLM 自动生成的规则集在性能上可媲美专家手工编写的规则。

#### 7. 优点
*   **精准归因**：解决了“大海捞针”式的检测难题，通过注意力机制锁定嫌疑文本，大幅提升了检测效率和准确度。
*   **高度灵活**：规则系统是可插拔、可定制的，用户可以根据具体业务逻辑（如白名单操作）调整安全策略。
*   **模型无关性**：该防御框架可以保护各种后端 LLM，包括闭源 API 模型。

#### 8. 不足与局限
*   **计算延迟**：由于引入了额外的 LLM 推理步骤，单次动作的检测延迟（约 8s）较高，可能不适用于对实时性要求极高的场景。
*   **调用策略依赖**：为了平衡成本，论文建议仅在执行“高风险工具调用”时触发监控，这依赖于用户对风险操作的准确定义。
*   **语义模糊风险**：在某些极端情况下，良性的复杂指令可能与注入指令特征重叠，导致误报（False Positive）。
